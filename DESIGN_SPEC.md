# 设计规范文档（Design Specification Document）

## 1. 设计概述

- **产品名称**：宥宝礼物兑换系统
- **设计理念**：打造温暖、鼓励、有趣的家庭积分管理体验，用卡通绘本风格让孩子感受成长的快乐，用简洁高效的界面帮助家长轻松记录
- **目标用户**：
  - 主要用户：3-12岁儿童（视觉激励为主，大字号、高对比度）
  - 次要用户：28-45岁家长（效率优先，操作简洁）
- **设计原则**：
  1. **童趣化但不幼稚**：使用圆角、柔和色彩，但保持界面清晰专业
  2. **视觉激励优先**：金色小元宝、礼花动画等元素强化正向反馈
  3. **双角色区分**：孩子端色彩活泼，家长端功能清晰
  4. **高效操作**：家长记账≤3次点击完成

## 2. 视觉设计规范

### 2.1 色彩系统

#### 主色调（孩子端为主）
- **天空蓝**：`#4A90E2`（主色调，用于界面背景、按钮、导航栏）
  - 浅色变体：`#6BA3E8`（悬停状态）
  - 深色变体：`#3A7BC8`（按下状态）
- **金黄色**：`#FFD700`（小元宝图标、积分数字、强调元素）
  - 浅色变体：`#FFE44D`（元宝光泽高光）
  - 深色变体：`#E6C200`（元宝阴影）

#### 辅助色彩
- **柔和粉**：`#FFB6C1`（女孩向装饰、次要按钮）
- **青草绿**：`#90EE90`（正向积分标签背景）
- **珊瑚橙**：`#FF7F50`（待审批状态、提示信息）

#### 中性色（通用）
- **深灰**：`#333333`（标题文字、重要信息）
- **中灰**：`#666666`（正文文字）
- **浅灰**：`#999999`（辅助文字、禁用状态文字）
- **边框灰**：`#E0E0E0`（分割线、输入框边框）
- **背景灰**：`#F5F5F5`（页面背景、卡片背景）
- **纯白**：`#FFFFFF`（卡片、弹窗背景）

#### 功能色彩
- **成功绿**：`#52C41A`（成功提示、批准操作）
- **警告橙**：`#FAAD14`（警告提示、待审批状态）
- **错误红**：`#F5222D`（错误提示、负分标签、拒绝操作）
- **信息蓝**：`#1890FF`（信息提示、链接）

#### 家长端色彩调整
- 使用更低饱和度的色彩，减少视觉刺激
- 主色调改为`#5B9BD5`（更专业的蓝）
- 移除粉色和橙色装饰，保持简洁

### 2.3 布局系统

#### 栅格系统
- **移动端（≤767px）**：单列布局，内容区宽度100%
- **平板端（768px-1024px）**：12列栅格，间距16px
- **桌面端（≥1025px）**：12列栅格，间距24px，最大宽度1200px居中

#### 间距规范（8px基础单位）
- **超小间距**：4px（图标与文字间距）
- **小间距**：8px（标签内边距、列表项垂直间距）
- **标准间距**：16px（卡片内边距、按钮高度内边距）
- **中等间距**：24px（卡片之间间距、模块间距）
- **大间距**：32px（页面顶部间距、分组间距）
- **超大间距**：48px（页面底部间距）

#### 圆角规范
- **小圆角**：4px（输入框、标签）
- **标准圆角**：8px（按钮、小卡片）
- **大圆角**：12px（大卡片、弹窗）
- **超大圆角**：20px（孩子端主要按钮、小元宝容器）
- **圆形**：50%（头像、图标按钮）

#### 阴影系统
- **轻阴影**：`0 1px 4px rgba(0,0,0,0.08)`（悬停状态）
- **标准阴影**：`0 2px 8px rgba(0,0,0,0.12)`（卡片、按钮）
- **深阴影**：`0 4px 16px rgba(0,0,0,0.16)`（弹窗、下拉菜单）
- **特殊阴影（小元宝发光）**：`0 4px 20px rgba(255,215,0,0.4)`

## 3. 交互设计规范

### 3.1 导航系统

#### 孩子端导航
- **底部标签栏（Tab Bar）**：
  - 位置：固定在屏幕底部，高度64px
  - 标签数量：3个（首页、礼物、记录）
  - 图标：32px x 32px，彩色卡通风格
  - 标签文字：14px，选中状态加粗+天空蓝色
  - 选中效果：图标放大1.1倍 + 颜色变化 + 底部2px金色下划线

#### 家长端导航
- **顶部导航栏**：
  - 位置：固定在屏幕顶部，高度56px
  - 背景：渐变色`linear-gradient(135deg, #5B9BD5, #4A90E2)`
  - 标题：居中显示，白色18px
  - 返回按钮：左侧24px图标，白色
  - 退出家长模式：右上角"退出"按钮，白色16px

#### 页面跳转
- **切换动画**：页面从右向左滑入（300ms缓动），返回时从左向右滑出
- **弹窗动画**：从屏幕中心放大淡入（200ms），关闭时缩小淡出

### 3.2 交互反馈

#### 按钮状态
- **默认状态**：标准阴影，背景色主色调
- **悬停状态（桌面端）**：阴影加深，背景色变浅10%，200ms过渡
- **按下状态**：阴影消失，背景色加深10%，缩小至98%（100ms）
- **禁用状态**：灰色背景`#D9D9D9`，灰色文字`#999999`，无阴影，无hover效果
- **加载状态**：显示旋转loading图标（16px），文字改为"加载中..."

#### 表单交互
- **输入框**：
  - 默认：边框1px `#E0E0E0`，背景白色
  - 聚焦：边框2px 天空蓝`#4A90E2`，添加蓝色阴影`0 0 0 3px rgba(74,144,226,0.1)`
  - 错误：边框2px 错误红`#F5222D`，下方显示12px红色错误提示
  - 成功：边框1px 成功绿`#52C41A`，右侧显示绿色对勾图标
- **下拉选择器**：
  - 默认样式同输入框
  - 展开时向下滑动展开（200ms），最多显示5.5个选项（暗示可滚动）
- **数字输入器（快速记账）**：
  - 大号输入框，数字字体48px，居中显示
  - 支持键盘输入和+/-按钮快速调整
  - 正数显示绿色，负数显示红色

#### 加载状态
- **页面加载**：顶部显示蓝色进度条（2px高），从左到右滑动
- **局部加载**：显示旋转的loading图标（金色小元宝图标旋转动画）
- **骨架屏**：在内容加载时显示灰色占位框，带渐变闪烁动画

#### 提示反馈
- **成功提示（Toast）**：
  - 位置：屏幕顶部居中，向下滑入
  - 样式：绿色背景，白色文字，左侧对勾图标
  - 显示时长：2秒自动消失
- **错误提示（Toast）**：
  - 样式：红色背景，白色文字，左侧叉号图标
  - 显示时长：3秒自动消失
- **确认对话框（Modal）**：
  - 从屏幕中心淡入放大（200ms）
  - 背景遮罩：黑色50%透明度
  - 弹窗：白色背景，圆角12px，最大宽度400px
  - 按钮：双按钮布局（取消/确定），取消在左，确定在右

---

### 3.3 动效规范

#### 过渡效果
- **页面切换**：300ms，使用`cubic-bezier(0.4, 0, 0.2, 1)`缓动函数
- **模态弹窗**：200ms，使用`ease-out`
- **下拉展开**：200ms，使用`ease-in-out`

#### 微交互
- **按钮点击**：缩放至98%（100ms），回弹至100%（200ms）
- **卡片悬停**：上浮4px + 阴影加深（200ms）
- **复选框/开关**：滑动动画200ms + 颜色渐变

#### 特殊动画
- **小元宝脉冲动画**：
  - 当积分增加时，小元宝图标放大至1.2倍（300ms），回弹至1.0倍（200ms）
  - 同时播放金色光圈扩散动画（600ms）
- **礼花动画**：
  - 兑换成功时，从屏幕中心向四周发射彩色粒子（3秒）
  - 使用Canvas或SVG实现，粒子颜色：金黄、粉色、青绿、天空蓝
  - 中央显示祝贺文字，带弹跳动画
- **日历格子点击**：
  - 点击时放大至1.05倍（100ms），回弹至1.0倍（150ms）
  - 边框颜色变为金黄色，带脉冲效果

## 6. 响应式设计

### 6.1 断点设置
- **移动端（手机）**：0 - 767px
- **平板端（Pad）**：768px - 1024px
- **桌面端**：≥1025px

### 6.2 布局适配

#### 移动端（≤767px）
- 单列布局，内容区宽度100%
- 礼物卡片：2列网格
- 日历格子：7列，正方形
- 底部标签栏：固定底部，3个标签
- 字号：使用移动端标准字号

#### 平板端（768px-1024px）
- 主要目标设备，优化重点
- 礼物卡片：3列网格
- 日历格子：7列，格子更大（更易点击）
- 左右内边距：24px
- 小元宝容器：放大至320px宽
- 家长端表单：单列居中，最大宽度600px

#### 桌面端（≥1025px）
- 内容区域最大宽度1200px，居中显示
- 礼物卡片：4列网格
- 孩子端：左侧日历仪表盘，右侧礼物推荐（2列布局）
- 家长端：左侧导航菜单（固定），右侧内容区（可滚动）
- 鼠标悬停效果：卡片上浮、按钮高亮

### 6.3 组件适配

#### 按钮
- 移动端：高度48px，全宽或最小宽度120px
- 平板/桌面端：高度44px，内容宽度自适应

#### 输入框
- 移动端：高度48px，字号16px（防止iOS自动缩放）
- 平板/桌面端：高度44px，字号14px

#### 卡片间距
- 移动端：16px
- 平板端：20px
- 桌面端：24px

#### 弹窗
- 移动端：全屏弹窗（从底部滑入）
- 平板/桌面端：居中弹窗，最大宽度600px

## 8. 开发交付说明

### 8.1 设计资产

#### 导出规范
- **图标**：SVG格式（矢量），或PNG格式（@1x, @2x, @3x）
- **插图**：SVG格式优先
- **元宝图标**：提供PNG格式（64px, 128px, 192px）
- **颜色变量**：提供CSS变量定义文件

#### 命名规范
- 图标：`icon-[功能名]-[尺寸].svg`，如`icon-home-24.svg`
- 插图：`illustration-[场景].svg`，如`illustration-empty-gift.svg`
- 元宝：`ingot-[尺寸].png`，如`ingot-64.png`

### 8.2 技术建议

#### 前端技术栈
- **框架**：Vue 3（轻量级，适合PWA）或原生JavaScript
- **CSS方案**：
  - CSS变量实现主题色系统
  - Flexbox + Grid布局
  - CSS动画实现微交互
- **图标**：
  - 使用Iconfont或SVG Sprite
  - 推荐库：Iconify（支持按需加载）
- **动画库**：
  - Lottie（礼花动画）
  - GSAP（复杂动画）
  - 或原生Canvas/SVG实现

#### PWA配置
- **manifest.json**：
  - 应用名称："宥宝礼物兑换系统"
  - 主题色：`#4A90E2`
  - 图标：提供192px、512px图标
- **Service Worker**：
  - 缓存策略：Cache First（静态资源）+ Network First（API请求）
  - 离线页面：提供离线提示页面

### 8.3 实现注意事项

#### 性能优化
1. **图片优化**：
   - 礼物图片使用WebP格式
   - 懒加载：首屏外图片延迟加载
   - 响应式图片：根据屏幕尺寸加载不同分辨率
2. **动画优化**：
   - 使用CSS transform（GPU加速）
   - 避免频繁reflow/repaint
   - 礼花动画使用requestAnimationFrame
3. **首屏优化**：
   - 关键CSS内联
   - 骨架屏占位
   - 字体子集化（仅加载常用汉字）

#### 交互细节
1. **触摸优化**：
   - 按钮最小点击区域：44px x 44px
   - 防止误触：添加300ms延迟取消（使用touch事件）
   - 滚动优化：使用`-webkit-overflow-scrolling: touch`
2. **表单优化**：
   - 数字输入框设置`type="number"`和`inputmode="numeric"`
   - 失焦时自动保存草稿
   - 实时校验，避免提交时才报错
3. **无障碍支持**：
   - 图片添加alt属性
   - 按钮添加aria-label
   - 键盘导航支持（Tab键）

#### 兼容性
- **iOS Safari**：
  - 测试输入框自动缩放问题（字号≥16px）
  - 测试PWA安装流程
  - 测试圆角和阴影兼容性
- **Android Chrome**：
  - 测试Service Worker缓存
  - 测试手势操作（滑动、捏合）
- **桌面浏览器**：
  - 测试鼠标悬停效果
  - 测试键盘快捷键
